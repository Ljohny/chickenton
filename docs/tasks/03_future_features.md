# Step 3: 향후 구현할 기능 (장기 계획)

이 문서는 SnapNote 기능 개발의 장기적인 로드맵을 정의합니다. `Step 2`의 핵심 기능 구현 및 리팩토링이 완료된 후 진행될 내용입니다.

## 예정된 기능

### 1. 게시글 CRUD (생성, 읽기, 수정, 삭제)

- **게시글 작성**: 사용자가 새로운 게시글을 작성하고 Firestore에 저장하는 기능을 구현합니다. (`create_post_screen.dart` 활성화)
- **게시글 수정**: 기존 게시글의 내용을 수정하는 기능을 추가합니다.
- **게시글 삭제**: 게시글을 삭제하는 기능을 구현합니다.

### 2. 사용자 상호작용 기능

- **공유하기**: 외부 앱으로 게시글 내용을 공유하는 기능을 추가합니다.
- **좋아요**: 사용자가 게시글에 '좋아요'를 누를 수 있는 기능을 구현하고, 좋아요 수를 집계합니다.
- **댓글**: 게시글에 댓글을 작성하고 볼 수 있는 기능을 구현합니다.

### 3. 상세 화면 기능 강화

- **더보기(...) 메뉴**: 상세 화면의 '더보기' 버튼을 눌렀을 때 '수정하기', '삭제하기', '공유하기' 등의 옵션을 제공하는 메뉴를 구현합니다.
- **타임스탬프 포맷팅**: `post_card.dart`에 남아있는 `TODO`를 해결하여, 게시글 작성 시간을 "5분 전", "2시간 전" 등과 같이 사용자 친화적인 형태로 표시합니다.

### 4. 사용자 인증

- Firebase Authentication을 연동하여 회원가입 및 로그인 기능을 구현합니다.
- 게시글 및 댓글이 작성자 정보와 연결되도록 데이터 모델을 확장합니다.

## 우선순위

1.  게시글 작성 기능 구현
2.  타임스탬프 포맷팅
3.  게시글 수정 및 삭제 기능 구현
4.  댓글 기능 구현
5.  좋아요 기능 구현
6.  사용자 인증 연동
7.  공유하기 기능 구현
